# CLI Reference

The `native` CLI provides project scaffolding, markup validation, automation, packaging, and debugging tools. Every verb accepts `--help` (prints usage and exits 0); unknown flags error with the usage text.

## Commands

<table>
  <thead>
    <tr>
      <th>Command</th>
      <th>Description</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><code>native init [path] [--frontend &lt;native|next|vite|react|svelte|vue&gt;] [--full]</code> (default: native)</td>
      <td>Scaffold a new Native SDK project with the specified frontend (<code>native</code> scaffolds a native-rendered markup app with no web frontend, and no build files — the CLI owns the build; <code>--full</code> also writes build.zig, build.zig.zon, editor config, and CI). Omit <code>path</code> to scaffold into the current directory.</td>
    </tr>
    <tr>
      <td><code>native dev [dir]</code></td>
      <td>Build and run the app in the current (or given) app directory — a Debug build by default (the markup hot-reload watcher and the Debug-only teaching diagnostics are compiled in only in Debug; pass <code>-Doptimize=...</code> to override), printing a one-line completion and naming any failing step; apps with a frontend dev config also get the managed dev server. With <code>--binary &lt;path&gt;</code>, only runs the frontend dev server flow against a prebuilt shell.</td>
    </tr>
    <tr>
      <td><code>native build [dir]</code></td>
      <td>Build a ReleaseFast binary into <code>zig-out/bin/</code>, printing a one-line completion and naming any failing step</td>
    </tr>
    <tr>
      <td><code>native test [dir]</code></td>
      <td>Run the app's test suite, printing the zig build summary (step/test tally) plus a final <code>native test: passed</code> line; it also refreshes the model-contract artifact <code>native check</code> reads</td>
    </tr>
    <tr>
      <td><code>native check [dir] [--strict]</code></td>
      <td>Validate <code>src/**.native</code> markup and <code>app.zon</code> without building; with a fresh <code>zig-out/model-contract.zon</code> (refreshed by <code>native test</code>) it also checks bindings, iterables, and message tags against your <code>Model</code>/<code>Msg</code> and warns on model state no view uses (<code>--strict</code> fails on warnings). Without the artifact it degrades to structural checking and says so loudly: "model contract: not yet built - bindings checked structurally only; run <code>native test</code> to enable typed checks". Markup accessibility findings are reported per-file in full, and a failing <code>src/*.native</code> file that no Zig source embeds gets a leftover-file hint</td>
    </tr>
    <tr>
      <td><code>native eject [dir]</code></td>
      <td>Write an owned <code>build.zig</code>/<code>build.zig.zon</code> into the app (once); the verbs then drive your files via <code>zig build</code></td>
    </tr>
    <tr>
      <td><code>native doctor</code></td>
      <td>Check host environment, WebView, manifest, and CEF</td>
    </tr>
    <tr>
      <td><code>native cef install</code></td>
      <td>Download, prepare, and verify the macOS CEF runtime</td>
    </tr>
    <tr>
      <td><code>native cef path</code></td>
      <td>Print the default or configured CEF directory</td>
    </tr>
    <tr>
      <td><code>native cef doctor</code></td>
      <td>Check only the CEF layout</td>
    </tr>
    <tr>
      <td><code>native validate [app.zon]</code></td>
      <td>Validate <code>app.zon</code> against the manifest schema</td>
    </tr>
    <tr>
      <td><code>native package</code></td>
      <td>Package the app for distribution</td>
    </tr>
    <tr>
      <td><code>native bundle-assets [app.zon] [assets] [output]</code></td>
      <td>Copy frontend assets into the build output</td>
    </tr>
    <tr>
      <td><code>native package-windows</code></td>
      <td>Package shortcut for Windows</td>
    </tr>
    <tr>
      <td><code>native package-linux</code></td>
      <td>Package shortcut for Linux</td>
    </tr>
    <tr>
      <td><code>native package-ios</code></td>
      <td>Package shortcut for iOS</td>
    </tr>
    <tr>
      <td><code>native package-android</code></td>
      <td>Package shortcut for Android</td>
    </tr>
    <tr>
      <td><code>native markup check &lt;file.native&gt; [more files...]</code></td>
      <td>Validate Native markup without building; errors carry <code>file:line:column</code> and a teaching message (files still on the former <code>.zml</code> extension check too, with a rename note); run inside an app directory it picks up a fresh model contract the same way <code>native check</code> does</td>
    </tr>
    <tr>
      <td><code>native markup lsp</code></td>
      <td>Run the markup language server (diagnostics, completion, hover) for editor integration</td>
    </tr>
    <tr>
      <td><code>native automate &lt;command&gt;</code></td>
      <td>Interact with the automation server</td>
    </tr>
    <tr>
      <td><code>native skills list</code></td>
      <td>List built-in AI agent skills</td>
    </tr>
    <tr>
      <td><code>native skills get &lt;name&gt; [--full]</code></td>
      <td>Output built-in AI agent skill content</td>
    </tr>
    <tr>
      <td><code>native version</code></td>
      <td>Print the native version</td>
    </tr>

</table>

## `native cef` flags

<table>
  <thead>
    <tr>
      <th>Flag</th>
      <th>Description</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><code>--dir</code></td>
      <td>CEF install directory. Defaults to the host platform directory under <code>third_party/cef</code>.</td>
    </tr>
    <tr>
      <td><code>--version</code></td>
      <td>CEF binary version to download. The default is Native SDK's pinned tested version.</td>
    </tr>
    <tr>
      <td><code>--source</code></td>
      <td><code>prepared</code> or <code>official</code>. Defaults to <code>prepared</code>, which downloads Native SDK's no-CMake runtime from GitHub Releases.</td>
    </tr>
    <tr>
      <td><code>--download-url</code></td>
      <td>Override the prepared runtime release base URL, or the official CEF host when using <code>--source official</code>.</td>
    </tr>
    <tr>
      <td><code>--allow-build-tools</code></td>
      <td>Allow the advanced official CEF path to invoke local build tools for <code>libcef_dll_wrapper</code>.</td>
    </tr>
    <tr>
      <td><code>--force</code></td>
      <td>Redownload and replace the target directory.</td>
    </tr>
  </tbody>
</table>

Core maintainers who need to build CEF before a Native SDK runtime release exists should use `tools/cef/build-from-source.sh`. The CLI's default `native cef install` path remains the no-CMake app-developer path.
