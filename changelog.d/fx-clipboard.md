feature: **Clipboard on the effects channel**: `fx.writeClipboard` / `fx.readClipboard` put text on (and read it from) the system clipboard through the platform pasteboard — the same `PlatformServices` seam the runtime's cmd+C copy uses — so TEA apps stop spawning `pbcopy`. Same discipline as spawn/fetch/file: key-based (shared key space and slots), exactly one terminal Msg with an explicit outcome (`ok` / `failed` / `rejected` / `cancelled`), bounded at the platform's 64 KiB clipboard limit (over-bound writes are rejected outright; over-bound reads fail whole, never cut), and fully fake-executor testable (`pendingClipboardAt`, `feedClipboardResult`). The pasteboard call runs synchronously on the loop thread — no worker thread — and the result still drains as an ordinary Msg. The effects-probe example grows a Copy-status button dogfooding the write path.
