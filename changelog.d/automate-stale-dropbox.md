fix: **`native automate` refuses stale dropboxes loudly**: every snapshot now carries the publishing app's pid (`publisher_pid=` in the header), and the CLI checks it against the live process table before trusting the file — `snapshot`/`list` error immediately, `wait` keeps polling for a LIVE publisher instead of returning yesterday's `ready=true`, and `assert` neither satisfies nor refutes patterns against a dead publisher's file. The error names the absolute path and teaches the fix (rebuild and relaunch with `-Dautomation=true`); pid-less snapshots from older builds are treated as stale too.
