perf: **Accessibility publish only when the tree changed**: `updateWidgetAccessibility` used to ride every owned display-list refresh — on a live host that is a full native-accessibility tree assembly and publish per refresh, paid multiple times per click and 60 times a second under animation with an unchanged tree. The runtime now fingerprints the exact node set it hands the platform (every field, string bytes length-prefixed, optional presence) and skips the platform call when the fingerprint matches the last successful publish; typing, focus moves, toggles, and scroll offsets change a node and republish immediately, while animation and hover churn that leaves the tree identical publishes nothing. A failed publish records no fingerprint, so the next refresh retries.
