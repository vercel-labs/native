fix: **Occluded windows no longer stop presenting**: a frame requested by the runtime now always completes regardless of window visibility — when macOS stops compositing an occluded/unfocused window and `CAMetalLayer` stops vending drawables, the AppKit host completes the frame from the retained canvas texture (paced at the display's refresh interval) instead of silently dropping it, and flushes the retained content to the glass when the window is next composited; `allowsNextDrawableTimeout` is disabled so a starved drawable pool can no longer stall the main thread for 1 s per frame.
- **GPU frame completions publish their facts**: a frame that resolves a pending input into a recorded input latency, or records the first-frame latency, now invalidates the runtime so automation snapshots republish — an occluded window's widget click no longer leaves the perf harness polling a snapshot frozen before the completion (steady-state frames still stay quiet).
