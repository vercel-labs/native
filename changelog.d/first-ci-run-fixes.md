fix: **Debug macOS apps no longer abort at launch**: Zig 0.16.0's Debug UBSan traps spuriously on the macOS SDK's inline `dispatch_once` (`__builtin_assume` in dispatch/once.h) and zig's ubsan_rt panics decoding the report, so any Debug-built example crashed with `Abort trap: 6` before its first frame — every ObjC host compile now passes `-fno-sanitize=builtin` (build helper, standalone examples, scaffold template).
- **Windows builds link again**: the standalone example builds and the scaffold template were missing `gdi32`/`imm32`, which the gpu_surface software present path (`SetDIBitsToDevice`/`StretchDIBits`) and the IME composition path (`ImmGetContext`/`ImmGetCompositionStringW`) in `webview2_host.cpp` need — `zig build -Dplatform=windows` failed with 7 lld-link undefined symbols.
- **Correct embed viewport floats on x86_64 Debug**: Zig 0.16.0's self-hosted x86_64 backend miscompiles the SysV C ABI for `zero_native_app_viewport`'s f32-heavy signature (safe-area insets arrived as garbage; reproduced on x86_64-linux, correct under `-fllvm`), so test artifacts and the mobile embed static library force the LLVM backend on x86_64 until the upstream backend is fixed.
- **`zero-native automate wait` now allows 30s** (was 5s) before declaring the app not ready, matching the `assert` default — a cold GTK/software-EGL launch on a loaded CI runner routinely needs more than 5s.
- **Smoke and perf harnesses dump the app log on failure**: every macOS smoke recipe and `scripts/perf-gpu-dashboard.sh` print the crashed/timed-out app's log (and the perf harness the dashboard-canvas snapshot line), so a CI failure carries its own forensics instead of a bare `automation snapshot was not ready`.
- **Runtime TypeScript contract check follows the split modules**: `check-runtime-types.js` read `runtime/root.zig` and `platform/root.zig` markers that moved to `runtime/bridge_responses.zig` and `platform/types.zig` in the module splits, so `npm run scripts:check` failed with phantom cursor/profile-risk drift.
