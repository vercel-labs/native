fix: **Big Models no longer blow the comptime eval-branch quota through the markup engines**: `UiApp`'s default features compile `MarkupView(Model)`, and its `collectItemTypes` scan died with `evaluation exceeded 1000 backwards branches` at ovation's ~200-pub-decl Model — pointing at framework code the app never asked to run. Every Model/Msg-scaled comptime walk in both engines now derives its quota from the scanned type (`typeScanQuota`: linear headroom per field/decl plus the item-type dedupe's worst-case quadratic term), so apps never raise `@setEvalBranchQuota` for framework scans and `runtime_markup = false` goes back to being a binary-size choice, not an escape hatch. Guarded by `ui_markup_huge_model_tests.zig`, an ovation-shaped fixture (~240 pub decls, 24 distinct slice item types) that must build through the interpreter, the compiled engine, and a default-features `UiApp` without any quota raise.
