feature: **Honest single-line text** — `ElementOptions.wrap = false` / markup `wrap="false"` on `text`: plain text leaves measured ONE line but painted with word wrap, so a width-constrained list-row title silently painted its second line over the row below; the only escape was a hand-built clip stack. `wrap` is now a tri-state line policy: `true` word-wraps through the span paragraph machinery as before, an explicit `false` renders one line clean-clipped to the frame the element receives (`TextWrap.none` + a frame clip — no ellipsis: the text machinery has no elide mode, and synthesized ellipsis bytes would diverge from the selection/copy source of truth), and unset keeps the classic path byte-identical. Measurement is unchanged and already single-line in both the intrinsic and constrained paths, so paint now agrees with layout; static-text selection geometry mirrors the painted line. Both markup engines, the validator vocabulary tests, the LSP attribute docs, the skill, and the docs cover the new value (the container `wrap` teaching error stands).
