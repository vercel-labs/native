perf: **Click response frames repaint the change, not the window**: a Msg-driven rebuild used to degrade dirty bounds to the full window — the presented summary records ids and bounds but not content, so a rebuild marked every keyed command changed even when the retained-patch diff knew only a handful differed, and the response frame of a toggle click presented as a full-window scissor (milliseconds of host redraw plus a full-surface texture upload). The frame planner now derives rebuild dirty bounds from the SAME key+fingerprint edit script the patch present ships — the union of every upserted command's old and new extents plus every evicted command's extent — and keeps the per-change rect clusters, so changes at opposite window corners present as a dirty rect LIST (binary packet v3, up to 8 rects behind a header flag) instead of their bounding union. The host clears, clips, culls, and uploads per rect: pixels between two far-apart changes stay retained, and a backdrop blur sitting between them is not recomputed at all. Refinement refuses conservatively (no retained baseline, unkeyable command list, a z-order shuffle of unchanged commands, small views) and falls back to the previous union; a no-op rebuild whose edit script is empty now presents nothing. The incremental-verify channel byte-compares every refined update against a full redraw, and the dirty-rect derivation is pinned by unit test.
- **Clipped commands join the per-command raster cache**: a command clip is a plain rect carried by the command, so clipped output is still a pure function of the command — the cache fill now applies the clip and shrinks the raster extent to bounds∩clip (scrolled content with mostly-offscreen bounds never rasterizes its full extent). Panel and scroll-view content, which dominates content-heavy views, now blits on unchanged redraws instead of re-shaping through the system text stack.
- **Backdrop blur is O(region), not O(region × radius)**: both box-blur passes keep the exact clamped-window average (byte-identical output) but slide the sample window incrementally instead of rescanning it per pixel — a repaint that covers a backdrop-blurred surface stops costing milliseconds of scalar resampling.
- `NATIVE_SDK_GPU_DRAW_TRACE=1` now attributes each present: commands drawn after culling, split into cache blits, fresh cache fills, and direct draws, with per-group time.
