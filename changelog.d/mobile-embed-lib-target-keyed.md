fix: **Mobile embed libraries stage per target — an interleaved iOS run can no longer poison the Android APK**: the embed static library the dev and package loops build now installs under `.native/embed/<target-triple>/` instead of one shared `zig-out/lib/lib<name>.a` slot, so the iOS simulator, iOS device, and Android builds each own a private stage and cross-target sessions cannot clobber each other (the shared slot let a Mach-O archive land where the Android host link looked, and a `-shared` link tolerates undefined symbols, so ld.lld pulled nothing, only warned, and the installed APK died at dlopen with `UnsatisfiedLinkError: cannot locate symbol "native_sdk_app_create"`). Two honesty guards back the keyed stage: the host compiles verify the staged archive's actual object format against the target before linking (a mismatch names the file, what it is, and the rebuild that fixes it), and the Android host link now passes `-Wl,--no-undefined` so any dangling reference fails at build time on the host instead of at launch on the device.
