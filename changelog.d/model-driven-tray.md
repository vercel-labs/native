feature: **Model-driven tray menu and title**: `UiApp.Options.status_item_fn` derives the menu-bar extra's title and dropdown from the model — consulted on install and after every rebuild, re-applied only when the output changed (title and menu patch independently), so an open-count badge in the title and a latest-items dropdown are one function. Selections dispatch each item's `command` through `on_command` with source `.tray`, exactly the window-menu shape; the static `status_item` keeps icon and tooltip. Title changes ride a new platform seam (`PlatformServices.updateTrayTitle` / `Runtime.updateTrayTitle`) that retitles the live macOS `NSStatusItem` button without re-creating it; platforms without the seam keep menu updates and log the title gap once. Fully fake-platform testable (`trayTitleUpdateCount`, recorded title/items).
