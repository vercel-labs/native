feature: **Anchored floating surfaces**: a widget with `layout.anchor` set (`ElementOptions.anchor`/`anchor_alignment`/`anchor_offset`, markup `anchor="below|above"` + `anchor-alignment` + `anchor-offset` on `dropdown-menu`) FLOATS against its parent's resolved frame instead of the flow — below/above with auto-flip at the window edges, `start`/`end`/`stretch` alignment (stretch widens to the trigger, the select-menu look), height clamped to the chosen side, zero flow space (siblings never reflow), rendered in a late z-pass above the whole tree, and window-clipped rather than cropped by scroll/clip ancestors. Hit-testing, focus visibility, and dirty bounds hoist identically, an open menu follows its scrolling trigger, and open state stays model-owned (the surface exists only while the view renders it). Budget: 16 anchored surfaces per view (`max_canvas_widget_anchored_per_view`, loud overflow).
- **Dismissal is a Msg the model owns** (`ElementOptions.on_dismiss`, markup `on-dismiss` on dialog/drawer/sheet/dropdown-menu): Escape, click-outside, and automation/accessibility dismiss now dispatch a `canvas_widget_dismiss` event that `UiApp` maps to the surface's handler — the engine's immediate hide is just the optimistic echo, and the next rebuild's source tree is truth. Escape on a focused trigger finds its own anchored menu, and clicking the trigger of an open surface toggles exactly once (no dismiss-then-reopen race). Apps without a handler keep the previous behavior byte for byte.
- **`select`/`combobox` pickers reach their honest conclusion**: still no `options=` attribute — the composition IS the data channel — but the sanctioned `stack` + `select` + if-gated `dropdown-menu` pattern now anchors for real (no clipping, no layout push) and closes model-side via `on-dismiss`. The markdown-viewer's toolbar sample picker is the shipped adoption.
- **Press-and-hold** (`ElementOptions.on_hold`, markup `on-hold`): a pointer held ~350 ms dispatches the hold Msg through the runtime timer path and the release presses nothing, a quick click dispatches `on-press` as usual, and a right/ctrl-click whose route offers no context menu dispatches the hold Msg immediately (declared context menus win) — dev-2's Menu+primaryAction shape for breadcrumb switchers.
- Both markup engines in parity (validator, interpreter, comptime compile errors), LSP docs/completions, skill + docs coverage, and the dismissible-surface kind set deduplicated into one `canvas.widgetKindDismissibleSurface` predicate.
