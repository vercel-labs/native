feature: **Context menus in Native markup — the OS menu by default, one authored form**: `<context-menu>` is a first-class markup element — a direct child of the pressable element it answers, holding `<menu-item on-press="trash_note:{n.id}">` items and `<separator/>`s with `if`/`else`/`for` to swap or repeat them — lowering to the same `ElementOptions.context_menu` channel Zig views use, so a right/ctrl-click presents the real OS menu (macOS `NSMenu`) and the selection dispatches its typed Msg. On hosts without a native menu presenter (Linux GTK, Windows Win32 today) the SAME declared items present automatically as an anchored canvas surface on the widget, with the standard Escape/outside-click dismissal: authors write one menu, the platform decides presentation, and a canvas imitation never renders where the OS menu exists.
- **Native menus are automation-drivable**: snapshots list every widget's declared items in invocation order (`context_menu=["Rename","Delete"]`, separators keep their slots, disabled items say so), and the new `widget-context-menu <view> <id> <item-index>` verb invokes an item by dispatching the same `context_menu_action` platform event a real pick produces — journaled and replayable — because the OS menu's tracking loop cannot be driven programmatically. Dead invocations fail by name (undeclared menu, out-of-range index, separator slot, disabled item). Protocol version bumps to 5.
- Total across the stack: validator and both engines reject misuse with teaching errors (a menu on a non-pressable host, a second menu, attributes on the menu, an item without `on-press` or with an `icon`, non-item children), `native check` types item payloads against the model contract, NSUI serializes the new vocabulary under a fresh registry code (additive — no schema-version bump), and the LSP/docs vocabulary gains completion, hover, and the platform-support matrix row.
- **Notes adopted the policy**: the folder/note/trash row menus are now `<context-menu>` declarations — real OS menus on macOS, same Msgs and flows — the model dropped its open-menu state entirely, the synthetic All Notes row declares an empty item set (so no menu presents), and the tests drive presentation with `widget-context-press` and selection with `widget-context-menu`, plus layout/a11y sweeps over the anchored fallback surface. The anchored `<dropdown-menu>` grammar remains for app-designed surfaces (select-style pickers, button dropdowns, hold-reveal menus).
