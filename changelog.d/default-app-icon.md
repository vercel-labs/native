improvement: **A real default app icon**: the old default was a full-bleed letterform tile at the root (and a 65-byte ASCII "placeholder" text file masquerading as `icon.icns` in 20 of the showcase apps — macOS fell back to the generic document glyph). The new default is a neutral layered-surface mark rendered from vector source (`tools/generate_app_icon.zig`) through the SDK's own path rasterizer onto the macOS icon grid — 1024 canvas, centered 824x824 rounded-rect plate with standard margins, baked drop shadow, token-adjacent blue-violet gradient — so it sits at the correct visual size in the Dock from 16 px to 1024 px. `zig build generate-icon` regenerates everything from source (the full @1x/@2x `.icns` family assembled and round-trip-validated by the built-in container writers, plus `icon.ico`, a 1024 px `icon.png`, and an `icon.svg` design mirror; no external tools, PNGs deflate-compressed in-tool), `native init` embeds the icon in the CLI so scaffolds get a real icon from any directory (the old fallback was an empty 8-byte header), and every showcase app now ships the new default.
