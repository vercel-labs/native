feature: **Markdown viewer example (`examples/markdown-viewer`)**: a showcase split-pane markdown editor/preview authored almost entirely in markup — a `textarea` elm-mirror on the left, one `<markdown>` element on the right bound to the same model bytes, so the preview tracks every keystroke. It exercises the honest no-dialogs file pattern (an editable toolbar path field driving Open / Save / Save As through `fx.readFile`/`fx.writeFile`, with every outcome surfaced in the status bar), a recent-files sidebar persisted to the per-app data dir through the same file effects, links opening in the system browser via `fx.spawn`, model-owned `<details>` expansion, a two-mode stone/indigo theme flipped through `tokens_fn` with `on_appearance` feeding the system scheme, and four bundled sample documents. `zig build test-example-markdown-viewer` covers the dispatch paths with the fake effect executor plus automation snapshot and widget-click assertions.
