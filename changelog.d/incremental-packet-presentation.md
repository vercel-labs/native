feature: **Incremental packet presentation — frame cost scales with what CHANGED, not view size**: the AppKit host now keeps a retained command dictionary per gpu-surface view (keyed by each command's ObjectId, or a synthetic index+fingerprint key for unkeyed commands), and steady-state binary presents ship a `patch` — evicted keys, re-encoded upserts for the commands whose content fingerprint changed, and the full draw-order vector — instead of the whole command list. On a 200-message chat-shaped transcript whose full binary present is ~73 KB, a hover/switch toggle patches in ~3.3 KB, a single text edit in ~3.5 KB, and a message append in ~3.6 KB (~20x less wire per frame); unchanged text runs are never re-encoded, so their per-frame `layoutTextRun` re-measure disappears too. Live in the showcase apps: the components-catalog switch toggle is a 1.1 KB patch, a notes keystroke ~2.4 KB, with `present_mode=patch` holding under interaction and zero fallbacks.
- **Drift can cost one full present, never wrong pixels**: full keyed presents carry a generation; a patch names the generation it edits, and ANY disagreement — retained state lost, generation mismatch, decode failure, retained-command budget (`max_canvas_retained_packet_commands_per_view`, pinned host-side as `NATIVE_SDK_PACKET_RETAINED_COMMAND_CAP`) — refuses the patch, and the engine re-presents FULL in the same frame under a fresh generation. JSON packets, scissor-subset loads, and raw pixel presents all invalidate the retained state on both sides. A patch that would re-encode every command (a scroll step moving the whole view) deliberately re-baselines with a full present instead of shipping a bigger-than-full patch.
- **Wire format v3** (`binary_packet_version = 3`; this patch machinery landed the v1→v2 restructure, and the dirty-rect list refinement bumped v2→v3 in the same release): every binary header gains the generation, every command rides behind an explicit retain key, and load action 3 is the patch edit script; generation 0 means "draw but do not retain". The host pins the layout independently and refuses any framing disagreement — never garbage on the glass.
- **Observable, not asserted**: snapshot view lines gain `present_mode=full|patch`, `present_patch_bytes`, `present_patch_upserts`, `present_patch_evicts`, and `present_retained_commands`; patch refusals extend the fallback-reason channel as `present_fallback=patch_refused` (transient — the same-frame full resync clears it, the running fallback-frame counter keeps the history).
- **Golden-tested at the wire-byte level**: a scripted interaction sequence (toggle, scroll step, text edit, message eviction) drives two engines — one patching, one full-presenting every frame — into Zig twins of the host's retained store, and every step's retained state must match byte-for-byte (command encodings AND draw order). Plus refusal→same-frame-resync, patch-overflow→full, JSON-invalidates-baseline, and a fingerprint field-coverage audit so a newly encoded field can never silently stop invalidating patches.
