feat: **Deterministic session record and replay**: a Native SDK session is literally its deterministic init, the platform-event sequence, and the effect results that crossed the effect boundary — and now it records that way. Launch any app with `NATIVE_SDK_SESSION_RECORD=<path>` (or `native automate record --out <path> -- <app>`) and the dispatch choke point journals every platform event, every drained effect result (fetch bodies, subprocess lines and exits, file and clipboard reads, journaled `fx.wallMs` clock reads), a state fingerprint per published frame, and a pixel hash at every automation screenshot. `native automate replay <path> --verify -- <app>` re-runs the journal headlessly — the fake executor parks every spawn/fetch/file/clipboard request and the journaled results are the only world — and verifies each checkpoint, failing loudly at the first divergence. The journal is a versioned length-prefixed binary with fixed budgets; truncated, tampered, cross-platform, and protocol-skewed files are refused with teaching errors. Same-platform replay is the v1 bar. Quits now seal the journal reliably: the macOS host delivers shutdown on app termination and treats SIGTERM as a graceful quit.
