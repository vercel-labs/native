feature: **Zero-config Zig toolchain**: when the pinned Zig (0.16.0) is missing from PATH, `native dev|build|test` offer to download the official build into `~/.native/toolchains/zig-0.16.0/` and use it from then on — explicit consent prompt in interactive mode, `--yes` for automation, and the archive is verified against SHA-256 checksums pinned in the CLI itself (recorded from the official release index), so the CLI version pins the framework + Zig pair end to end. A PATH Zig with a compatible version (same major.minor, at least the pinned patch) is always preferred, and `NATIVE_SDK_ZIG` overrides resolution entirely.
