perf: **Accessibility publishes come off the input-to-glass path**: the platform accessibility publish (a full native tree assembly on live macOS, the single largest fixed cost a click paid before its pixels moved) used to run inside the input dispatch, ahead of the responding present. A publish requested during an input dispatch or a gesture's refresh batch — and only one whose fingerprint says the tree actually changed — now defers to after that input's presented frame (same tick, post-present); semantics consumers tolerate milliseconds, the glass should not. Deferrals with no repaint in flight settle synchronously at dispatch end (there is no present to protect), accessibility-action dispatches force-flush before returning so the AX client that initiated the action reads the post-action tree, and direct API callers publish synchronously as always.
