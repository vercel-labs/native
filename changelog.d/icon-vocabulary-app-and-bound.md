feature: **App icons and bound icon names in markup**: icon values grew two forms beyond the closed built-in vocabulary — `app:<name>` draws an icon the app registered with `canvas.icons.registerAppIcons` (declare the table as `pub const app_icons` so the model contract carries the names and `native check` verifies every `app:` reference with a did-you-mean; without a fresh artifact the check degrades to structural checking and says so), and `icon="{binding}"` makes the choice model data (a per-row status icon, the soundboard's play/pause toggle — one button, no if/else arms). Both engines lower the forms identically (parity-tested), NSUI round-trips them as plain attribute values under the existing codes (no schema bump), and the LSP completes built-in names plus the `app:` prefix inside icon-valued quotes.
- **Honest icon failure mode**: a non-empty icon name that resolves nowhere at draw time now renders a missing-icon fallback (a slashed circle) with a Debug warning naming the value — a broken bound name or unregistered `app:` reference is visible and loud, never a silent gap (`Ui.appIcon` rides the same channel).
