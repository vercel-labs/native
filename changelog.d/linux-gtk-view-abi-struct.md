improvement: **Linux GTK host takes view create/update by descriptor pointer**: `native_sdk_gtk_create_view` and `native_sdk_gtk_update_view` are now reached through a single `native_sdk_gtk_view_desc_t`/`native_sdk_gtk_view_patch_t` pointer instead of 20+ positional arguments, so the Zig platform layer never crosses the wide mixed C-ABI signature that Zig 0.16.0's self-hosted x86_64 backend mis-lowers (stack-passed args land shifted). This removes the arg-placement hazard at the seam — the fault the x86_64 LLVM-force in the build graph works around — as a step toward letting the self-hosted backend build Debug on x86_64.
