fix: **Escape dismisses anchored surfaces opened from non-focusable triggers**: the dismissal search was rooted at the focused widget, so a menu opened from a plain-text trigger — which takes no focus on click — left Escape dead while outside-click and `widget-action dismiss` worked. Escape now falls back to the topmost MOUNTED anchored surface (highest in tree order, matching the late z-pass the user sees on top) whenever the focus chain yields none, dispatching the same `on_dismiss` Msg through the same contract; a focused editable with live IME composition still wins — Escape cancels the composition and never dismisses, not even through the fallback.
