fix: **Dev runs fall back to the embedded default Dock icon**: an unbundled macOS run (`native dev`, a plain `zig-out` binary) whose configured icon file does not exist now renders the toolkit's embedded default icon into the Dock — the same fallback `native package` ships — instead of the generic executable tile. The showcase apps no longer carry per-app copies of the default `icon.icns` (26 of them had drifted to a stale older design); apps that ship no icon of their own now have exactly one source of truth, and `zig build generate-icon` also regenerates the notes example's raw one-image icon source in lockstep so the packaging mask/inset demo can never go stale either.
