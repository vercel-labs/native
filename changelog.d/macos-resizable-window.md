fix: **`resizable = false` is honored on macOS**: app.zon carried the flag, the runtime carried it, and the AppKit host dropped it at the C ABI — `NSWindowStyleMaskResizable` was hardcoded for every window. Both create entry points (`native_sdk_appkit_create` for the main window, `native_sdk_appkit_create_window` for secondaries) now take the flag and build the style mask conditionally, with the CEF host carrying the same symbols in sync; the zoom button disables automatically on fixed windows. The null platform records the flag per created window so the runtime-to-platform propagation is regression-tested (shell windows included). Verified while there: `title`, size, position-under-restore, and `restore_state` are honored; still ignored deliberately — `restore_policy` (no backend reads it; macOS always clamps to the visible screen) and GTK/Win32 resizability (their own ABIs also lack the flag; scoped out deliberately).
