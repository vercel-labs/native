feature: **Docs Components section with engine-rendered previews**: the docs site gains a full Components section — an index grid plus one page per built-in component (34 pages: button through timeline, including the 45-icon gallery and the builder-only chart) — where every preview is a light/dark webp pair drawn OFFSCREEN by the deterministic reference renderer, the same pixels the engine ships. One command regenerates everything: `zig build docs-component-previews` renders every scene at 2x (including an engine-hover state via a synthetic pointer move), shells out to lossless `cwebp`, and also dumps the markup vocabulary (`docs/src/lib/component-vocab.json`) straight from the LSP doc tables and validator name lists — the docs' `<AttrTable>` resolves rows from that JSON and fails the build on any name the validator doesn't know, so attribute tables cannot drift from the grammar. Every `.zml` snippet on the pages passes `native markup check`; the LSP doc tables moved to `tools/native-sdk/markup_docs.zig` (re-exported from `markup_lsp.zig`) so the generator can import them without module collisions; the old Built-in Components page now points at the new section.
