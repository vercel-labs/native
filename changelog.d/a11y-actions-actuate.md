fix: **Accessibility actions now actuate instead of reporting success on nothing**: performing AXPress (or any assistive action) on a canvas widget now rides the same key-driven activation the automation widget verbs use, so widgets wired through message handlers — an `on-press` tab, an album card, an `on_toggle` checkbox, a slider's steps — respond to assistive technology exactly like they respond to a keyboard user, in every build. The macOS bridge also stops advertising actions an element cannot perform (a static label no longer offers press/increment/decrement/cancel), and writing AXFocused now moves the app's real focus instead of flipping a flag the app never hears.
