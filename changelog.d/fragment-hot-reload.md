feature: **Fragment hot reload**: hybrid apps — a Zig builder root embedding compiled markup fragments — now keep the dev edit-see loop. Register each fragment with `View.fragment("src/header.native")` in `UiApp.Options.fragment_watch`, and in Debug dev runs the markup watch polls every registered file plus every file each fragment's imports reach: editing a file reloads exactly the fragments it serves (a shared component file reloads all of its dependents in one poll) with the same degrade family as the single-root watch — a bad save keeps the last good view and records the file:line teaching diagnostic, the next good save recovers, and a byte-for-byte revert drops back to the compiled path. Release binaries carry none of it: the registration handle is empty outside Debug (no source paths, no polling), and the snapshot header's `markup_watch=armed|off` stays honest — a hybrid app reports armed in Debug and off in release.
