perf: **The widget-tree reconcile now scales with what changed, not with view size**: every rebuild matched the incoming tree against retained state (control echoes, source-wins selection/slider/scroll semantics, text selections, source list semantics) and diffed it for invalidations with per-node linear scans — control entries, source-control entries, source semantics, text entries, and the invalidation diff's id lookups plus its duplicate-id validation were all O(n^2) at the 1024-node budget, and the retained copy re-paid the same scans a second time. The reconcile lookups now ride the same deterministic probe-table index the frame planners adopted (`canvas.plan_key_index`): lookups return exactly what the scans returned (the lowest-index entry matching the same predicate), so reconciled trees, invalidations, and every source-wins contract are byte-identical while the searches collapse to O(n). Small views skip the index entirely (below 64 entries the scans win), keeping the tiny-view floor untouched, and inputs larger than the runtime budgets fall back to the scans.
- Measured with `zig build bench-render -Doptimize=ReleaseFast`: `reconcile` p50 drops ~3.8x on the big-view toggle (the top engine stage on Msg-driven rebuilds), ~1.9x on the transcript scroll, and ~2x on first-frame; end-to-end toggle p50 improves ~1.4x with `chart-tick` unchanged.
