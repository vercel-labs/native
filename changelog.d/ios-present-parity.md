improvement: **iOS presentation reaches macOS-path parity**: the toolkit-owned UIKit host now presents the deterministic CPU-rendered frame the way the macOS host does — the RGBA8 pixels upload straight into a sampled Metal texture drawn by the same fullscreen-quad presenter pipeline (the render pass converts to the drawable's BGRA, deleting the per-frame CPU swizzle and its second staging buffer), the drawable is acquired only on frames that will really present and only after the frame's CPU work is paid (the canvas-revision gate already pins an idle app at zero drawable acquisitions and zero upload bytes), and the pump follows the scene lifecycle like the macOS host follows window occlusion: the display link pauses in the background, resign/become-active forward runtime deactivate/activate (the Android onPause/onResume mirror), and returning to the foreground re-presents the retained canvas exactly once even when the revision is unchanged, because the drawable pool may have been purged while covered. `NATIVE_SDK_GPU_FRAME_TRACE=1` now prints the macOS-style per-present trace on iOS too (drawable vend time, upload bytes) plus a once-per-second structural summary — the counter form of the idle law. Rendering is unchanged: frames still come from the deterministic CPU reference renderer, and the embed pixel ABI carries no damage rects yet, so a changed frame re-renders and re-uploads the full surface.
