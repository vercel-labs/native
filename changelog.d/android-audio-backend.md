feature: **Android audio playback and streaming**: the toolkit-owned Android host registers the platform audio service over the embed ABI (`native_sdk_app_set_audio_service` + `native_sdk_app_audio_event`) with one `android.media.MediaPlayer` behind it — local files and verified cache entries prepare synchronously with the real decoded duration, URL sources stream progressively with honest `buffering` reports, ~500ms position ticks arrive only while playing, one completion fires at natural end, and a parallel cache-filling download (part file, size-verified against the manifest, atomic rename) lands in the app data directory's `cache/audio/` (`app_dirs` now resolves Android directories inside the per-app data dir, which the host exports as `HOME`/`TMPDIR`). The host owns honest audio focus: requested on play, and a focus loss pauses the transport and reports the paused state through the ordinary position event stream (no auto-resume, no media-session or notification integration yet). The host APK gains the `INTERNET` permission with a network security policy that keeps TLS required on the internet and scopes cleartext to local networking only (localhost and the emulator's `10.0.2.2` host alias, for the dev loop's local media server).
