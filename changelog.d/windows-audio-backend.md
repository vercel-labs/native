feature: **Windows audio playback and streaming**: the Win32 host gains the full audio effect family through a Media Foundation media session (source resolver into the Streaming Audio Renderer — everything ships with Windows 10/11), matching the macOS contract exactly: local MP3 playback, progressive HTTP(S) URL streaming with honest `buffering` reports, pause/resume, sample-accurate seek, per-player volume, ~500ms position ticks only while playing, and one completion at natural end. URL sources resolve verified-cache-first, then stream while a parallel WinHTTP download fills the shared track cache (part file, size-verified against the manifest, atomic rename — a partial file never occupies the cache name), so the second play of a track is local. `audio_playback` and `audio_streaming` now report supported on the Windows system-WebView host; the soundboard and deck examples play for real on Windows.
