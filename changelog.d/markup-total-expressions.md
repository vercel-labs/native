feature: **Total expressions in ZML bindings**: `{...}` now takes a pure, total expression grammar — arithmetic (`+ - * /`, division always a float, division by zero a loud teaching error), comparisons (`== != < <= > >=`), boolean logic (`and`/`or`/`not`), `++` string concatenation, parenthesization, literals, and a closed 16-function formatting library (`fixed`, `thousands`, `percent`, `date`/`time`/`datetime` in UTC from model timestamps, `upper`/`lower`/`trim`, `min`/`max`/`abs`, `round`/`floor`/`ceil`, `plural`) — in text interpolation, attribute values, `if` tests, and template args; message tags, `each` iterables, and import paths stay path-only.
- **One evaluator, both engines**: the interpreter and the comptime-compiled engine resolve bindings their own way and evaluate through one shared core, so results are bit-for-bit identical, floats included; type mismatches the model types make knowable are compile errors in the compiled engine with the same message the interpreter teaches at build time.
- **Checkable without the model**: `native markup check` validates expression syntax, complexity bounds (256 bytes / 64 terms / 16 levels, taught one past), function names with did-you-mean, arity, and literal type discipline; `now()` is a teaching error naming the model/fx loop as the clock's home, and expression string literals ride the tofu guard.
