feature: **The tall titlebar variant**: `titlebar = "hidden_inset_tall"` (app.zon, ShellWindow, window descriptors) is the tall unified-toolbar look — the same hidden-inset shape with the unified-toolbar-height band (~52pt), where macOS vertically centers the traffic lights, so a toolbar-height header row and the lights share a centerline instead of the lights parking ~10pt high. Implemented as an empty borderless `NSToolbar` + `NSWindowToolbarStyleUnified` + `titlebarSeparatorStyle = .none` (pure geometry: nothing drawn, drags/click-through/double-click zoom all verified live); in fullscreen the host hides the toolbar (it would otherwise pin a blank band over the app's header) and a `contentLayoutRect` KVO re-emits resizes so the chrome re-query always reads the settled band — enter and exit both live-verified pixel-exact. GTK/Win32 keep standard chrome, same scoping as `hidden_inset`.
- **`on_chrome` now carries the whole chrome truth**: the hook's payload is `platform.WindowChrome` — the overlay `insets` plus the traffic-light cluster's `buttons` frame in content coordinates (top-left origin), so a header can pad AND vertically center against the lights instead of assuming the band midpoint. The null platform serves a settable `window_chrome` for tests; `examples/markdown-viewer` ships the tall retrofit (its toolbar row tracks the band height, falling back to its natural 48 in fullscreen).
