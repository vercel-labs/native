feature: **Windowed virtual lists**: `ui.virtualWindow` + `ui.virtualList` are the first-class infinite-scroll primitive — the runtime owns the viewport math (retained scroll offset + viewport → visible index range from a fixed per-item extent), the model owns the data, and the view builds ONLY the visible window plus overscan, so widget budgets stay viewport-sized at 100,000 items. The list is a runtime-scrolled region: engine wheel/kinetic/keyboard scrolling applies everywhere, the native scroll driver takes over on macOS with its content size set to the FULL virtual extent (the scrollbar always tells the truth), scroll observations re-derive the view with no app wiring, offsets clamp against the virtual extent, and rows keyed by item identity keep their structural ids — and their engine/model state — across window shifts. First builds and window-growing resizes converge within one rebuild via a coverage retry against the fresh geometry. Legacy `virtualized` containers (no declared count) keep their model-driven contract byte-identically.
- **`on_reach_end` — the approach-end signal**: any scroll container (markup: `on-reach-end` on `scroll`, both engines + validator + LSP docs) dispatches one Msg when a user scroll comes within one viewport of the content end, with hysteresis — it re-arms only past 1.5 viewports, which appending a batch causes on its own by growing the extent, so infinite fetch is one Msg per approach and never a storm.
- **`examples/feed`** — the proof: a 100,000-post deterministic timeline (posts derive from their index; no network) riding one windowed virtual list with reach-end batching, per-post likes/boosts keyed by post index that survive scrolling away and back, and a test suite driving real dispatch: window-only builds, identity across shifts, reach-end hysteresis, and snapshot telemetry showing `widget_nodes` viewport-sized while the scroll semantics report the full 8.4M-point extent.
- The docs grew a Virtual List component page (with a live windowed preview — the wasm host now re-derives scenes on scroll like the app loop), and the native-ui skill documents the pattern, the data-window seam, and the markup exclusion call (the closed grammar has no channel for a `for` binding to receive the runtime's range request, so the windowed list is builder-only; markup keeps bounded `<list virtualized>` plus `on-reach-end`).
