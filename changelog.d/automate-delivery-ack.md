fix: **`native automate` waits for delivery**: the command dropbox is a single slot, and consecutive commands could overwrite an unconsumed predecessor while still printing "queued". The CLI now waits for the app's `done` ack (one command per presented frame) and prints `delivered <action>`; an unconsumed command is a loud teaching error, never a silent drop.
- **Stale-instance guard**: automate verbs warn loudly when the publishing app started before the newest binary in `zig-out/bin` was built — an older instance impersonating the new build cost whole misdiagnosis rounds.
- **`native dev` owns its process tree**: dev children run in their own process groups and the whole tree is killed on exit and on SIGINT/SIGTERM/SIGHUP, so a killed dev session can no longer leave an orphaned app publishing snapshots.
- **Wheel failures teach**: a `widget-wheel` on a missing/non-interactive/zero-size target lands in the snapshot as `WheelTargetUnknown`/`WheelTargetNotInteractive`/`WheelTargetHasEmptyBounds` with the command arguments as `detail=`, instead of a blanket `InvalidCommand`; `automate wait` timing out with no snapshot now names `-Dautomation=true` and the working directory as the usual causes.
