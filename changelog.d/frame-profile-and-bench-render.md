feature: **Per-stage frame profiling (`native automate profile on|off`) and a render macro-benchmark (`zig build bench-render`)**: the runtime now stamps cheap monotonic timings around every frame-pipeline stage — view rebuild, layout, widget reconcile, display-list emit, accessibility publish, frame plan, patch derivation, wire encode, the host present call, and the macOS host's packet decode/draw split — into rolling p50/p90 windows that are zero-cost while disabled (one bool per stamp site, no allocation, percentiles sorted only at snapshot time). While profiling is on, the automation snapshot carries a greppable `frame_profile <stage>_p50_us=... <stage>_p90_us=... <stage>_max_us=... <stage>_n=...` line (automation protocol bumped to v3 for the new verb).
- `zig build bench-render -Doptimize=ReleaseFast` runs six deterministic scenarios through the real pipeline (null platform, binary retained/patch packets): first frame on a ~930-widget view, keystroke into a focused field, a switch toggle driving a full TEA rebuild, wheel steps through a 200-message markdown transcript, a timer-driven chart tick, and a README-sized markdown re-render — reporting end-to-end p50/p90 per interaction plus the per-stage breakdown, with noisy rows flagged.
