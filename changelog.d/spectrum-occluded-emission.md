fix: **Spectrum reports pause while nobody can see the glass**: `.spectrum` bands describe a display, so hosts that know the app has no visible window stop emitting them — the 25 Hz band events no longer wake the update/rebuild loop for occluded glass, which is what regressed occluded-playing CPU from the ~3-4% class into the ~14% class after real spectrum analysis landed. macOS parks the analysis tick (no FFT, no event) while every host window reports occlusion-invisible (minimized, fully covered, hidden, inactive Space); Windows drops emission while every top-level window is minimized (the same IsIconic fact its frame heartbeat trusts); Linux/GTK keeps emitting, deliberately and documented, because no visibility fact holds across compositors. Playback and the position/loaded/completed reports continue untouched, the journal records the occluded stretch as honest silence (replay shows what a viewer could have seen — nothing), and emission resumes within one ~40 ms report of reveal, so analyzer glass like the deck's snaps current the moment it is back on screen. The null platform models the rule through per-window modeled occlusion (`setWindowOccluded`, and the minimize/focus verbs flip it), pinned by the audio-effects suite.
