feature: **The UI schema registry — one vocabulary, as data**: every markup vocabulary fact the stack used to keep in seven hand-synced lists (element names, option attributes, events, text-leaf/hit-target/stacking/dismissible/anchorable/icon-attr predicate sets, the rendered-text attribute set, token and icon vocabularies) now derives at comptime from one registry (`canvas.ui_schema`) with stable u16 codes assigned at birth and never reused. The validator, both markup engines, the contract checker, the LSP, and the CLI's did-you-mean all consume it; composite elements (markdown, stepper, timeline) attach their bespoke validation by named rule hook, so vocabulary is data while judgment stays code. The old lockstep tests became registry↔engine conformance tests, plus a fingerprint test that pins codes against silent renumbering.
- **Byte-range spans**: every markup node, attribute name, attribute value, and text run now carries its exact byte range into the parsed document (the write-back prerequisite); diagnostics keep their line:column shape, and a conformance test holds those positions equal to the span-derived ones.
- **The typed document pass**: attribute values and interpolated text runs now parse ONCE into typed forms (literal/binding/equality/expression-tree/message) at document level instead of per use per frame — both engines consume the typed document, the parity suites pin zero behavior change, and the interpreter stops re-parsing expressions on every frame.
- **NSUI, the canonical serialized document**: a deterministic schema-versioned binary encoding of a resolved view (registry codes on the wire, never names; flag-gated spans and provenance) with a document hash over the span-stripped structural content — reformatting or moving files never changes it, any structural edit does. Readers refuse unknown versions and codes loudly, never skip. `native markup dump <file.zml> [--out doc.nsui]` prints a JSON inspection view derived from the decoded binary.
- **Structural ids now hash stable kind codes, not enum ordinals**: `WidgetKind` gained a birth-assigned code table (`canvas.widgetKindCode`, frozen from the declaration order at the switch so every existing id is preserved), `structuralId` hashes it, and golden tests pin the full table plus exact id values — reordering the enum can never again renumber retained state, automation targets, or journal anchors.
