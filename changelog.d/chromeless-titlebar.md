feature: **Chromeless titlebar style**: a new explicit `titlebar = "chromeless"` window style for fully-skinned apps — NO OS chrome at all (macOS drops to a borderless key-capable window, Windows to the caption-less `WS_POPUP` shape, Linux to `gtk_window_set_decorated(FALSE)`), with window-drag regions still working on every desktop. An explicit opt-in with the affordance-honesty rule documented at the enum: apps that do not draw their own working window controls should keep the hidden styles, which keep the real OS buttons.
