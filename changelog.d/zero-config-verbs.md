feature: **Zero-config app verbs**: `native dev|build|test|check` work in an app directory holding only app.zon + src/ (+ assets) — no build.zig required. When the app has no build.zig the CLI synthesizes the same ~5-line `addApp` build graph into a gitignored `<app>/.native/build/` (Zig resolves path dependencies and `.zig-cache` relative to the build root, so the graph survives app moves and one deleted directory fully resets derived state) and drives it with `zig build --build-file`; apps that own a build.zig — including every example — are driven through plain `zig build` with zero behavior change. `dev` builds and runs with markup hot reload (and still owns the frontend dev-server flow for WebView apps; the legacy `native dev --binary` shape is untouched), `build` produces a ReleaseFast binary in zig-out/bin/, `test` runs the app suite, `check` validates src/**.zml plus app.zon without building, and `-D` flags (platform, web-engine, automation, optimize) forward straight to `zig build` because `addApp` in build/app.zig stays the single source of truth for module wiring.
- **`native eject`**: writes the fully-supported build.zig/build.zig.zon pair into the app exactly once — with a header explaining the ownership transfer — and refuses if either file exists; the verbs then drive the ejected build and never regenerate it.
- **Slim scaffold**: `native init` now emits just app.zon + src/ + assets + README (teaching the native verbs) with `.native/`, `zig-out/`, and `.zig-cache/` gitignored; `--full` keeps the previous shape (build.zig, build.zig.zon, .vscode, CI workflow), and web-frontend templates always scaffold full.
