feature: **App-parsed icon draw seam (`canvas.icons.registerAppIcons`)**: apps can comptime-parse their own SVGs in the common 24x24 stroke-icon dialect (`canvas.svg_icon.parseComptime`) and register the table once at boot; every widget draw path (icon leaves, buttons, icon buttons) resolves names through built-ins-first `icons.resolve`, so a registered app icon renders exactly like a built-in. `ui.appIcon(options, name)` is the builder entry (no comptime check — registrations are runtime facts; unknown names draw nothing with a Debug warning). Markup keeps the closed built-in vocabulary deliberately: the compiled engine validates icon names at comptime, where a runtime registration cannot exist, and the engines stay in strict parity — app icons are a Zig-view feature, documented as such.
