feature: **Variable-extent windowed virtual lists (v2)**: `ui.virtualList` no longer requires uniform rows — set `VirtualListOptions.extent_estimate` (a cheap pure fn from item index to an estimated extent: line counts, byte counts, any model fact) and rows lay out at their intrinsic wrapped heights while an engine-owned offset table (estimate prefix sums patched by MEASURED extents for every row that mounts) prices everything off screen. Scroll geometry — the scrollbar, scroll semantics, and the macOS native driver's content size — converges to measured truth as the user scrolls, and every correction is anchored on the first visible row: the offset absorbs the shift, so visible content never jumps (a scroll-storm suite asserts row positions to the point across hundreds of wheel steps, reversals, and deep jumps). Chat transcripts, mixed-height feeds, and markdown-bearing lists now ride the same viewport-sized budgets as uniform feeds.
- **Tail anchoring and history loading — the chat contract**: `anchor = .trailing` opens at the bottom and stays pinned through appends while the user sits at the bottom (a scrolled-away viewport is never yanked); `on_reach_start` mirrors `on_reach_end` (one Msg within a viewport of the content start, hysteresis re-arms past 1.5) for loading older history; and `index_base` gives items prepend-stable LOGICAL indices — decrease it by the prepended count and row identity, measured extents, and the viewport anchor all survive, with the offset growing by the prepended extent so the reader never moves.
- **Budgeted, canvas-limits style**: measured corrections are bounded per list (the farthest-from-anchor entry evicts when full and that row drifts back to its estimate until revisited — documented drift, no failure mode), estimate prefix sums are chunk-cached with a covered-range budget, and offset tables follow the existing per-build window budget.
- **`examples/feed` is now the mixed-height showcase**: one-liners to long-form walls, the estimate derived from body byte length, plus the zero-jump storm test. The uniform fast path (`item_extent > 0`) is byte-identical to before, legacy `virtualized` containers untouched, and the markup form keeps its documented exclusion (the closed grammar has no binding form for the estimate fn; builder-only, both modes).
