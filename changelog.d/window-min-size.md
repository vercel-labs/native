feature: **Window-enforced min sizes** — `min_width`/`min_height` on `WindowDescriptor`, `ShellWindow`, app.zon windows, and `platform.WindowOptions`: layout floors alone let a resizable window shrink below its honest minimum and clamp/clip its panes; the window now knows the floor the framework already knows. The declaration threads to the macOS host as `contentMinSize` (applied right after create, for the startup window and every runtime-created window alike; content size, chrome excluded), the null platform records `window_min_width`/`window_min_height` at the create seam for tests, and other hosts ignore the fields until they grow the channel. app.zon validation follows the titlebar pattern: negative or non-finite floors are a manifest error (`zig build validate`) and a comptime error in the app runner; 0 stays the "no floor" sentinel. The floor does not grow an already-smaller restored frame — it only stops future resizes.
