feature: **Real audio spectrum analysis** (`audio_spectrum`): hosts that can reach the player's PCM now analyze the app's OWN playback and deliver `.spectrum` audio events through the active playback's ordinary `on_event` Msg — 32 band magnitudes (log-spaced 50 Hz–16 kHz buckets, peak bin per bucket, each byte linear-in-dB from the −60 dBFS floor to full scale) at a steady ~25 Hz while audio is audibly playing; pause, stop, and a buffering stall starve the stream, so everything derived from the bands freezes honestly. The events journal verbatim at the effect boundary like position ticks (journal format v3), so record/replay repaints identical bars, the null platform ships a deterministic fake generator (`audioSpectrum`) so tests stay hermetic, and the automation snapshot's audio line carries `spectrum_events` + `spectrum_bands` as machine-checkable live evidence. Hosts that cannot analyze report the `audio_spectrum` feature unsupported and simply never send the events — honest absence, never fabricated bands.
- **macOS**: an `MTAudioProcessingTap` on the app's single player feeding a vDSP FFT — and local files UNIFY onto the AVPlayer streams always used (one tappable path covering local files, verified cache entries, and progressive streams), with a throwaway AVAudioPlayer probe preserving the synchronous decode verdict the cache-discard and no-URL-retry behaviors pin. The tap sits pre-effects: the bands describe the decoded track, not the volume fader.
- **Windows**: process-scoped WASAPI loopback capture of THIS app's audio session only (`ActivateAudioInterfaceAsync` process-loopback activation, include-process-tree; never system-wide, so other apps' audio can never pollute the bands) plus a hand-rolled in-box radix-2 FFT; requires Windows 10 2004+, probed live.
- **Linux**: GStreamer's `spectrum` element installed as the playbin's `audio-filter` (gst-plugins-good, probed live like the playbin itself), its linear magnitude messages folded into the documented log buckets on the loop thread.
- **The deck's analyzer is now real**: `spectrumLevels` draws the journaled band reports through model-side analyzer ballistics (instant attack on report arrival, linear decay on the frame clock — deterministic per journal), pause freezes the bars on real data, stop rests the glass, and an idle deck — or a host without `audio_spectrum` — shows the fixed noise-floor comb instead of fake dancing.
