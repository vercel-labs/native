feature: **Vector icons (`<icon name="..."/>` + `Ui.icon`)**: an SVG icon-subset parser (`canvas.svg_icon` — viewBox, path with the full `d` grammar including arcs, circle/ellipse/rect/line/polyline/polygon, fill/stroke/stroke-width/linecap/linejoin, `currentColor`, `<g>` inheritance; comptime- or runtime-parseable, std-only, fixed budgets) plus a curated built-in set of 20 stroke icons (24x24 stroke dialect) authored for the framework (`canvas.icons`: search, plus, x, check, the four chevrons, menu, settings, trash, edit, copy, external-link, play, pause, info, alert, arrow-right, download). Icons lower to wire path elements at comptime, draw as transform-wrapped `stroke_path`/`fill_path` commands (anti-aliased in the reference renderer via the vector core, native path rendering through the AppKit packet path), tint through the `foreground` color token (`currentColor`), and scale crisply to any size.
- **Markup element**: `<icon name="search"/>` in both engines with a closed, validated name vocabulary — a typo is a compile error in the compiled engine and a positioned teaching error in the interpreter, the validator, and the LSP. Zig views use `ui.icon(.{ .width = 16, .height = 16 }, "search")` (comptime-checked) or `ui.iconGlyph` for the historical text-glyph icons, which keep working unchanged.
- **Stock icon-set compatibility**: real, unmodified files in the common 24x24 stroke-icon dialect parse and rasterize through the same pipeline (verified against stock `rocket` and `circle-check-big` shapes, comptime and runtime); an app can `canvas.svg_icon.parseComptime(@embedFile("assets/icons/rocket.svg"))` its own icons. Not full SVG by design: no CSS, gradients, filters, text, transforms, or animation.
