feature: **One-image app icons**: drop a single square image in your project — `assets/icon.png` (1:1, ideally 1024x1024) or `assets/icon.svg` — list it in app.zon `.icons`, and `native package` generates what every platform needs, with no external tools (not even `iconutil`): a complete macOS `.icns` (16-1024 px, 1x + 2x) with the platform icon convention applied automatically — full-bleed artwork is inset on the icon-grid canvas (824/1024 artwork square, 185.4/1024 corner radius) and masked by the standard rounded rectangle, so a plain square image looks native in the Dock, while artwork with transparent corners is detected as pre-shaped and ships unmasked; a multi-size Windows `.ico`; a Linux hicolor PNG size set; an iOS asset-catalog icon; and Android launcher mipmaps. A prebuilt `.icns` (macOS) or `.ico` (Windows) in `.icons` always wins untouched for art-directed control, and packaging falls back to the SDK default icon when nothing is configured.
- **Quality**: every raster resize is an exact area-average (box) downscale over premultiplied linear-light values — 1024 to 16 px stays crisp instead of moire — and SVG sources rasterize per output size through the vector core; the mask composite is alpha-correct. The `.icns` and `.ico` containers are written directly (PNG payloads) and round-trip-validated by the pipeline's own parsers.
- **Teaching errors**: a non-square source errors with the found dimensions, an undecodable/unsupported file errors naming the two accepted forms, and a source under 512 px warns that upscaling never looks good — the same messages from `native validate`, `native check`, and packaging.
- `native init` now scaffolds `assets/icon.png` (the SDK mark rendered from vector source) instead of an opaque prebuilt `.icns`, so a fresh project demonstrates the contract end to end; `native doctor` reports the built-in icon pipeline instead of probing for `iconutil`, and `zig build generate-icon` assembles the default `.icns`/`.ico` with the built-in writers on any host OS.
