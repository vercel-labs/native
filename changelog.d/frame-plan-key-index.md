perf: **Frame planning now scales with what changed, not with view size**: the per-frame cache planners (render resources, text layouts, glyph atlas), the keyed display-list diff, and the presented-summary diff all resolved keys with linear scans, making every planned frame O(n^2) in commands — a one-character edit on a ~930-widget view re-paid the full quadratic cost. They now share a deterministic probe-table index (`canvas.plan_key_index`) that returns exactly what the scans returned (the lowest-index equal entry), so planner outputs, cache actions, and emitted packets are byte-identical while the searches collapse to O(n). Small views skip the index entirely (below 64 entries the scans win), keeping the tiny-view floor untouched, and inputs larger than the runtime budgets fall back to the scans.
- Measured with `zig build bench-render -Doptimize=ReleaseFast`: `plan` p50 drops ~6x on the big-view scenarios (keystroke, toggle) and the transcript scroll, `emit` roughly halves from the diff fix, and end-to-end interaction p50 improves ~2.3-3.2x with `chart-tick` unchanged.
