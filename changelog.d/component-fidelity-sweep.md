improvement: **Built-in components moved to the refreshed house design language.** The default `ColorTokens` palettes moved to the neutral house preset with the blue-violet primary (light `#1447e6`, dark `#193cb8`, both converted from the oklch source values and documented per token): neutral gray surfaces and text, translucent-white hairlines in dark mode (10% borders, with the 15% input wash as the pressed surface), dark cards one step lighter than the background, a mid-gray ring token, and high-contrast themes that keep the same primary at accessible depths. Existing apps' custom theme overrides are untouched — only the defaults moved.
- **Control geometry matches the reference metrics**: 36px default control height (buttons, inputs, select triggers), a borderless 44x24 switch pill with a 20px thumb inset 2px (knob-travel animation now shares the renderer's exact size- and density-scaled metrics), 16px checkbox with a literal 4px radius, 16px radio whose checked state is a primary dot over the unchanged input border, a 6px slider track with a 16px primary-ringed thumb, and a progress track that is the primary at 20% alpha instead of gray.
- **State colors follow the token scale**: filled primary/destructive buttons hover at 90% alpha (secondary at 80%), tabs (`segmented_control`) select by elevation — the active segment lifts to the page surface with a hairline border while inactive segments sit on the muted wash with muted text — and the accent-filled treatment stays one control-token override away.
- **Focus is a ring-offset ring**: every focusable control keeps its own border and strokes a concentric ring-token ring 2px outside it, instead of recoloring the border; invalidation bounds inflate by the same shared offset.
- **The search field's magnifier is a real circle**: the hand-drawn line box (visibly square at field sizes) is replaced by the registry `search` vector icon, whose circle survives small sizes because it flattens in viewBox units before the icon transform scales it down.
- Goldens (charts, markdown reference render, gpu-dashboard and gpu-components reference snapshots) were regenerated deliberately after before/after pixel review at 1x and 2x in light, dark, and high contrast.
