feature: **Search fields carry a built-in trailing clear affordance**: whenever a `search-field` holds text, a small `x` renders inside its trailing edge and a press on it clears through the standard text-edit path (selection and composition reset exactly like Escape's clear) — zero new attributes, exactly like the leading glass. Text layout, caret math, and the clip rect reserve the slot through the shared trailing-inset metric so text never runs under the glyph; the pointer edit is stamped onto the widget pointer event and `UiApp` maps it to the field's `on_input` Msg, so model-owned buffers clear too. Showcase apps dropped their external Clear buttons — the component carries it now.
