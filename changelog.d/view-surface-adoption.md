feature: **Native-surface adoption**: `Runtime.adoptViewSurface(window_id, label, handle)` installs an app-owned platform view (macOS: any NSView* the app constructed — a `VZVirtualMachineView`, an `MKMapView`) as the fill content of a declared native view container, generalizing the containment shape webview-backed child views already used. The platform keeps the adopted surface sized to its container across shell relayout and drops the bookkeeping when the container closes; `releaseViewSurface` detaches without destroying the app's view. Honest support reporting via `supports(.view_surface_adoption)` — macOS system host only today; Chromium/GTK/Win32/null reject explicitly. First consumer: `tools/guest-mac`, an in-repo macOS guest VM host (Virtualization.framework, no third-party VM software) whose windowed face is a Native SDK app hosting the live guest display through this channel, with headless `fetch/install/start/stop/status/ip` verbs so agents run live-GUI phases inside the guest instead of on the desktop.
